Krono — MetaModule plugin **v1.0.2** (Dan / 4ms feedback: faceplate, overlay, packaging).

Aligns with MetaModule graphics requirements (240px-tall faceplate PNG), removes the dynamic panel layer on device that covered the faceplate, and ships a single faceplate image in the `.mmplugin` tarball. VCV Rack manifest remains aligned with **krono_vcv** `plugin.json` (see that repo for the exact version string).

## Highlights

* **Faceplate:** `assets/KronoPanel.png` at **240px** height (SDK `graphics.md`); regenerate with `metamodule-plugin-sdk/scripts/SvgToPng.py --height=240` on `krono_vcv/res/KronoPanel.svg`, or `scripts/export_mm_faceplate_png.py` (Pillow fallback from `KronoPanel.jpg`).
* **Overlay:** On `METAMODULE` builds, `KronoPanelBackground` is no longer created — only `setPanel` + static PNG (see **krono_vcv** `KronoWidget`).
* **Package:** Only one panel raster in the plugin root (`KronoPanel.png`); do not add a duplicate under `assets/res/` (saves RAM on hardware).
* **SvgToPng / Inkscape:** `metamodule-plugin-sdk/scripts/actions/png.py` now invokes Inkscape without `shell=True`, so paths with spaces (e.g. `Program Files`) work.

## Hardware testing

**Not verified on physical MetaModule hardware** when this note was written. Please report issues at [krono_metamodule issues](https://github.com/sitar93/krono_metamodule/issues).
